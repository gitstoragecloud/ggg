## Stellar 概述

### 歷史背景
Stellar 是一個開源的去中心化支付網絡，由 Jed McCaleb 於 2014 年創立。值得注意的是，Jed McCaleb 曾擔任 Ripple 的首席技術官（CTO），在離開 Ripple 後創建了 Stellar Lumens。這一背景使得 Stellar 與 Ripple 有著相似的基因，但卻朝著不同的方向發展。

### 核心使命
Stellar 的核心使命是：
- **促進跨境支付**：提供快速、低成本的國際匯款服務
- **金融普惠**：為無銀行賬戶和銀行服務不足的地區提供金融服務
- **連接金融系統**：作為橋樑連接傳統金融機構與數字資產世界
- **開放網絡**：允許任何人在其上構建金融產品

### 組織結構
Stellar 由 Stellar Development Foundation（SDF）管理，這是一個**非營利組織**，與 Ripple 的營利性公司結構形成鮮明對比。這種非營利性質使 Stellar 更專注於普惠金融和公共利益。

## 技術架構

### Stellar Consensus Protocol (SCP)

Stellar 採用獨特的 **Stellar Consensus Protocol（SCP）**，這是一種基於 Federated Byzantine Agreement（FBA）的共識機制，也被稱為 **Proof-of-Agreement（PoA）**。

#### 核心特點

**1. 信任基礎系統**
- 不同於工作量證明（PoW）依賴計算能力
- 不同於權益證明（PoS）依賴質押權力
- SCP 依賴於受信任節點的協議
- 每個參與的 Stellar Core 節點可以決定信任哪些其他節點

**2. 開放成員資格**
- 任何人都可以成為 Core 節點
- 用戶定義的信任關係提供靈活性
- 去中心化控制，沒有中央權威

**3. 共識過程**
每個共識輪次分為兩個階段：

- **提名協議（Nomination Protocol）**：
  - 選擇候選交易集以納入帳本
  - 一旦節點確認其第一個候選，就停止投票提名新的交易集

- **投票協議（Ballot Protocol）**：
  - 確保網絡能夠一致確認和應用提名的交易集
  - 節點可以在確認候選後立即開始投票協議

#### 性能優勢

- **高速度**：據報導可處理高達 **1,000 次網絡操作/秒**
- **低成本**：交易費用極低
- **高能效**：比許多其他區塊鏈更節能
- **快速確認**：智能合約具有 **5 秒確認時間**
- **無挖礦獎勵**：網絡驗證者沒有金錢獎勵

#### 2025 年擴展目標

根據 Stellar 的 2025 年路線圖，預計通過技術改進將 Stellar Core 的理論吞吐量提升至 **5,000 TPS**。

## 代幣經濟學

### 供應量歷史

**初始供應**：
- 原始總供應量：1,000 億 XLM

**2019 年大燒毀**：
- 2019 年 11 月，Stellar Development Foundation 做出了大膽舉措
- 銷毀了約 **550 億 XLM 代幣**
- 將總供應量從 1,050 億（包括通脹）減少到 **500 億 XLM**
- 這是加密貨幣歷史上最大規模的代幣銷毀之一

### 當前供應結構

- **總供應量**：50,000,000,000 XLM（固定）
- **流通供應量**：約 27,000,000,000 XLM（2025年）
- **SDF 持有量**：約 22,000,000,000 XLM
  - 用於生態系統補助金
  - 合作夥伴關係
  - 用戶獲取
  - 運營成本

### 初始分配（基於 100 億）

1. **50%** - 分配給註冊 Stellar 帳戶的個人
2. **25%** - 分配給專注於增長和採用 Stellar 網絡的非營利組織和企業
3. **20%** - 分配給比特幣和 Ripple 持有者
4. **5%** - 分配給基金會運營資金

### 代幣用途

XLM（Lumens）在 Stellar 網絡中有多種用途：

1. **交易費用**：支付所有網絡交易費用
2. **最低餘額**：帳戶必須持有最低 XLM 餘額
3. **防垃圾郵件**：通過費用機制防止網絡垃圾交易
4. **橋接貨幣**：在不同法定貨幣之間充當中間貨幣

### 通脹機制

- **Stellar 沒有持續通脹或區塊獎勵**
- 供應量在 2019 年燒毀後固定
- 不像某些區塊鏈持續增發代幣

## Soroban 智能合約平台

### 概述

Soroban 是整合到現有 Stellar 區塊鏈中的智能合約平台，代表了 Stellar **十年歷史中最重要的升級之一**。

### 技術特點

**1. 編程語言**
- 合約使用 **Rust 語言**編寫
- 編譯為 **WebAssembly (Wasm)** 進行部署
- 提供強大的類型安全和性能優勢

**2. 性能指標**
- **5 秒智能合約確定性**
- 當前實時 TPS：**150 TPS**
- 2025 年目標：**5,000 TPS**

**3. 核心設計原則**
- **性能（Performance）**：快速執行和高吞吐量
- **可持續性（Sustainability）**：低能耗和成本
- **安全性（Security）**：內置安全機制和審計

### 發展時間線

- **2022年**：Soroban 項目啟動
- **2023年**：在 Futurenet 測試網上線
- **2024年3月**：主網第二階段上線
  - 經過兩年測試期
  - 允許創建用戶就緒的去中心化應用程序

### 開發者支持

**Soroban 採用基金**
- Stellar Development Foundation 推出 **1 億美元 Soroban 採用基金**
- 已資助超過 **160 個項目**
- 旨在鼓勵和支持開發者在 Stellar 上構建

### 企業應用

Soroban 特別設計用於：
- **DeFi 應用**：去中心化金融協議
- **企業開發**：大規模商業應用
- **資產代幣化**：現實世界資產上鏈
- **條件托管交易**：如房地產分期付款

這一合作關係使 Stellar 成為企業級跨境支付解決方案的重要參與者。

## Stellar vs Ripple 對比

由於 Stellar 和 Ripple 的共同歷史淵源，對比這兩個項目非常重要。

### 技術差異

| 特性 | Stellar (XLM) | Ripple (XRP) |
|------|---------------|--------------|
| **共識機制** | Stellar Consensus Protocol (SCP) - Federated Byzantine Agreement | Ripple Protocol Consensus Algorithm (RPCA) - 驗證者驗證交易 |
| **最大供應量** | 50 億（2019 年燒毀後） | 1,000 億 |
| **通脹/通縮** | 固定供應（燒毀後） | 通縮（燒毀交易費） |
| **交易速度** | 3-5 秒 | 3-5 秒 |
| **TPS** | 當前 1,000+，目標 5,000 | 1,500+ |
| **智能合約** | Soroban（Rust/Wasm） | 有限的智能合約功能 |

### 組織結構

**Stellar**
- **組織類型**：非營利組織（Stellar Development Foundation）
- **控制**：去中心化
- **代幣分配**：更去中心化的分配
- **透明度**：開源，社區驅動

**Ripple**
- **組織類型**：營利性公司（Ripple Labs）
- **控制**：更中心化
- **代幣分配**：Ripple Labs 控制大量 XRP
- **透明度**：企業結構

### 目標市場

**Stellar**
- **主要目標**：個人用戶和小型企業
- **重點地區**：新興市場
- **用例**：
  - 個人匯款
  - 普惠金融
  - 小額支付
  - P2P 轉賬
- **特點**：開放網絡，任何人都可以構建金融產品

**Ripple**
- **主要目標**：金融機構和銀行
- **重點地區**：已建立的金融市場
- **用例**：
  - 機構間結算
  - 銀行跨境支付
  - 流動性管理
  - 企業財資服務
- **特點**：為金融機構提供專業化解決方案

### 獨特功能

**Stellar 獨有**
- **內置 DEX**：具有內置訂單簿的去中心化交易所
- **多簽名支持**：原生多簽名功能
- **資產發行**：簡化的代幣發行流程
- **Soroban**：完整的智能合約平台

**Ripple 獨有**
- **RippleNet**：專為金融機構設計的封閉網絡
- **ODL（按需流動性）**：即時流動性解決方案
- **企業集成**：深度銀行系統集成

### 監管狀況

**Stellar**
- 作為非營利組織，監管壓力較小
- 更專注於合規和與監管機構合作
- CBDC 項目展示了與政府的合作

**Ripple**
- 面臨 SEC 訴訟（關於 XRP 證券地位）
- 2023 年法院裁決部分有利於 Ripple
- 持續的監管不確定性

### 社區與哲學

**Stellar**
- **哲學**：金融普惠，去中心化
- **社區**：開源社區驅動
- **發展**：公共利益優先

**Ripple**
- **哲學**：效率、企業解決方案
- **社區**：更專注於企業合作夥伴
- **發展**：商業利益驅動

### 相似之處

1. **共同創始人**：Jed McCaleb 創立了兩者（先 Ripple，後 Stellar）
2. **核心目的**：都專注於跨境支付
3. **速度和成本**：都提供快速、低成本的交易
4. **橋接貨幣**：原生代幣都可作為橋接貨幣
